= Installation
:navtitle: Installation
:description: How to install the Antora Dark Theme in your documentation project.

There are several ways to add the dark theme to your Antora project.
For most sites, use the *release UI bundle*—not the npm package.

== Which installation path should I use?

[cols="2,3", options="header"]
|===
| Goal | Method

| Ship dark docs with minimal setup (recommended)
| *Method 1* — pre-built `ui-bundle.zip`

| Custom logo, favicon, or a few supplemental files
| *Method 1* + one `supplemental_files` directory (<<overrides-on-bundle>>)

| Theme version in `package.json` and many custom partials on Default UI
| *Method 2* — npm (or GitHub) supplemental overlay

| Fork or maintain your own copy of theme assets
| *Method 5* — vendor `supplemental-ui`
|===

IMPORTANT: Antora accepts *one* supplemental directory *or* a virtual-file array—not a YAML list of multiple directories. Do not point `supplemental_files` at both `node_modules/antora-dark-theme/...` and another folder; merge overrides into a single path or use the release bundle plus a small override directory.

The https://www.npmjs.com/package/antora-dark-theme[npm package] publishes the same `supplemental-ui` tree for Method 2 and for theme development. It is not required when you consume the release bundle.

== Method 1: Pre-built UI bundle (recommended)

Each https://github.com/antora-supplemental/antora-dark-theme/releases[GitHub Release] ships `ui-bundle.zip`: Antora Default UI plus this theme, ready for your playbook.

. Configure `antora-playbook.yml`:
+
[source,yaml]
----
ui:
  bundle:
    # Pin a version for reproducible CI
    url: https://github.com/antora-supplemental/antora-dark-theme/releases/download/v1.0.9/ui-bundle.zip
    snapshot: true
----

. Build:
+
[source,bash]
----
antora antora-playbook.yml
----

You only need `antora` (or your existing docs toolchain)—not `npm install antora-dark-theme`.

TIP: Prefer `releases/download/vX.Y.Z/ui-bundle.zip` over `releases/latest/...` in CI so builds stay reproducible.

[[overrides-on-bundle]]
=== Overrides on the release bundle

Keep branding or small UI tweaks in *one* directory (for example `supplemental-ui-overrides/` with only `img/logo.svg`):

[source,yaml]
----
ui:
  bundle:
    url: https://github.com/antora-supplemental/antora-dark-theme/releases/download/v1.0.9/ui-bundle.zip
    snapshot: true
  supplemental_files: ./supplemental-ui-overrides
----

Antora overlays those files onto the bundle. You do not need to copy the full theme from npm.

See also: xref:configuration.adoc#virtual-supplemental-files[virtual supplemental files] for single-file entries.

== Method 2: npm registry (advanced)

Install when the theme must appear in your Node lockfile and you overlay `supplemental-ui` on the *Antora Default UI* zip (not the pre-built theme bundle).

[tabs]
====
pnpm::
+
[source,bash]
----
pnpm add -D antora-dark-theme
----

npm::
+
[source,bash]
----
npm install --save-dev antora-dark-theme
----

yarn::
+
[source,bash]
----
yarn add -D antora-dark-theme
----

bun::
+
[source,bash]
----
bun add -D antora-dark-theme
----
====

[source,yaml]
----
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/HEAD/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true
  supplemental_files: ./node_modules/antora-dark-theme/supplemental-ui
----

[[override-theme]]
=== Adding your own supplemental UI on top of npm

Antora’s `ui.supplemental_files` accepts *either* a single filesystem directory *or* an array of https://docs.antora.org/antora/latest/playbook/ui-supplemental-files[virtual files] (`path` + `contents`).
You cannot list multiple directories in the YAML array.

*Merged directory:* Copy `node_modules/antora-dark-theme/supplemental-ui` into your project (for example `supplemental-ui/`), merge your `partials/`, `css/`, and other assets, then set `supplemental_files` to that *one* path:

[source,yaml]
----
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/HEAD/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true
  supplemental_files: ./supplemental-ui
----

Prefer *Method 1* (release bundle) plus a small override directory unless you have a specific reason to stay on Default UI + npm supplemental.

== Method 3: Install from GitHub as a package dependency

Useful for an unreleased commit or when you prefer the `github:` protocol:

[tabs]
====
pnpm::
+
[source,bash]
----
pnpm add -D github:antora-supplemental/antora-dark-theme

# Specific tag
pnpm add -D github:antora-supplemental/antora-dark-theme#v1.0.9
----

npm::
+
[source,bash]
----
npm install --save-dev github:antora-supplemental/antora-dark-theme
----

yarn::
+
[source,bash]
----
yarn add -D github:antora-supplemental/antora-dark-theme
----
====

Use the same playbook `supplemental_files` path as *Method 2* (`./node_modules/antora-dark-theme/supplemental-ui`) with the Default UI bundle URL.

== Method 4: Install from release tarball

Download a specific `.tgz` from link:https://github.com/antora-supplemental/antora-dark-theme/releases[GitHub Releases] if you need an exact version without using the npm registry:

[tabs]
====
pnpm::
+
[source,bash]
----
pnpm add -D https://github.com/antora-supplemental/antora-dark-theme/releases/download/v1.0.9/antora-dark-theme-1.0.9.tgz
----

npm::
+
[source,bash]
----
npm install --save-dev https://github.com/antora-supplemental/antora-dark-theme/releases/download/v1.0.9/antora-dark-theme-1.0.9.tgz
----
====

Then use the same playbook `supplemental_files` path as *Method 2*.

== Method 5: Copy `supplemental-ui` folder

For maximum control, copy the `supplemental-ui` folder into your project:

. Clone or download the repository
. Copy the `supplemental-ui` folder to your project root:
+
[source,bash]
----
cp -r supplemental-ui /path/to/your/antora-project/
----

. Configure your `antora-playbook.yml`:
+
[source,yaml]
----
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/HEAD/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true
  supplemental_files: ./supplemental-ui
----

== Verify installation

Build your site to verify the installation:

[source,bash]
----
antora antora-playbook.yml
----

Open the generated site in a browser.
You should see a sun/moon toggle button in the navbar.
Click it to switch between light and dark modes.
