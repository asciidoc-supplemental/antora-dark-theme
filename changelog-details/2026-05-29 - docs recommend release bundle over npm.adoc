= 2026-05-29 — Bundle-first habit; npm deprecated

== Summary

Documentation now treats the GitHub release `ui-bundle.zip` as the *better solution* and the install habit to standardize on—not a soft “recommended default.”

The npm package is explicitly *deprecated*; npm publishing is planned to end. Discovery should move to the Antora Supplemental docs portal (extensions registry) instead of npmjs.

== Consumer guidance

* New sites: pinned `ui.bundle.url` to `releases/download/vX.Y.Z/ui-bundle.zip` only.
* Existing npm consumers: xref in guide to `npm-deprecation.adoc` (migrate, remove dependency, delete sync-from-`node_modules` scripts).
* Overrides: one `supplemental-ui-overrides/` (or virtual files)—never stack multiple supplemental directories.

== Package metadata

`package.json` includes npm’s `deprecated` message pointing at the bundle URL and https://antora-supplemental.github.io/docs/[Supplemental docs portal].

== Maintainer follow-up

* Stop advertising npm in README / installation.
* Remove npm publish from `.github/workflows/release.yml` once downstream migration is acceptable.
* Register the theme in `antora-extensions-registry` (via Supplemental docs) with bundle URL, not `npm install`.

== Not in this change

No removal of npm publish from CI yet; no registry unpublish—policy and docs only.
